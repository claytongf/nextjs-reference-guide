Next.js 16 uses the **App Router**: routes are defined by folders inside `app/`,
and special files describe each segment.

## File conventions

<FileTree
  tree={`app/
  layout.tsx      → shared UI that wraps every page
  page.tsx        → the / route
  about/
    page.tsx      → the /about route
  blog/
    [slug]/
      page.tsx    → dynamic /blog/:slug route`}
/>

A `page.tsx` makes a route public. A `layout.tsx` wraps pages in that segment
and persists across navigation.

## A page is a Server Component by default

```tsx
// app/page.tsx
export default function Home() {
  return <h1>Welcome</h1>;
}
```

This renders on the server. You only reach for `"use client"` when you need
interactivity — state, effects, or browser APIs.

<Callout type="info" title="Run it">
  Start the dev server with `npm run dev` and open
  <code>http://localhost:3000</code>. Editing `app/page.tsx` hot-reloads
  instantly.
</Callout>
